### 2.2 Phase 2: 向量索引 (3周) ⬆️ 延长 1周

**时间**: Week 3-5  
**目标**: 100K 向量存储与检索

#### 为什么延长?

1. **HNSW 实现复杂度**: 图构建算法需精细调参
2. **LSH 验证**: 需要大量测试验证假阳性率
3. **内存优化**: 需处理 400MB+ 内存管理

#### 任务清单

| 任务 | 工时 | 依赖 | 交付物 |
|------|------|------|--------|
| Flat Vector Storage | 2d | Phase 1 | `src/vector/flat-storage.js` |
| LSH 索引实现 | 3d | 设计文档 | `src/vector/lsh-index.js` |
| LSH 假阳性率测试 | 2d | LSH 实现 | `src/test/lsh-collision-sim.js` ✅ |
| HNSW 图构建 | 4d | - | `src/vector/hnsw-index.js` |
| 双层索引集成 | 3d | LSH+HNSW | `src/vector/combined-index.js` |
| 内存压力测试 | 2d | 全部 | Android 13 测试报告 |
| 性能基准 | 2d | 全部 | 性能测试报告 |

#### 新增风险处理

```
风险: HNSW 内存占用 400MB+ 可能导致 Android OOM
缓解: 
  - 已提供内存估算校正报告 (DEBT-HNSW-001-FIX)
  - 实施内存监控和警告
  - 准备 float16 量化降级方案
```

### 2.3 Phase 3: 同步机制 (3周) ⬆️ 延长 1周

**时间**: Week 6-8  
**目标**: 完整的设备间同步能力

#### 为什么延长?

1. **WebRTC 降级策略**: 需完整状态机实现
2. **ICE 失败处理**: 多场景测试 (NAT 类型识别)
3. **文件导出格式**: .hajimi 格式设计实现

#### 任务清单

| 任务 | 工时 | 依赖 | 交付物 |
|------|------|------|--------|
| Manifest 格式设计 | 2d | - | `src/sync/manifest.js` |
| Delta 算法实现 | 3d | Manifest | `src/sync/delta.js` |
| 文件导出/导入 | 3d | Delta | `src/sync/file-export.js` |
| WebRTC 基础连接 | 3d | - | `src/sync/webrtc.js` |
| **降级策略实现** | 3d | WebRTC+文件 | `src/sync/fallback-manager.js` ✅ |
| WebRTC 多场景测试 | 3d | 全部 | NAT 穿透测试报告 |
| 集成测试 | 3d | 全部 | 端到端同步测试 |

#### 关键交付: 降级策略

```
原设计: WebRTC P2P 为主
修正后: 文件导出为主，WebRTC 为辅助加速

理由:
- 文件导出 100% 可靠
- WebRTC 成功率依赖网络环境 (~30-95%)
- 自动降级保证用户体验
```
