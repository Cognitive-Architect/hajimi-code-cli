## 2. 理论分析

### 2.1 LSH 参数

```javascript
const LSH_CONFIG = {
  hashFunction: 'simhash-64',
  numTables: 8,       // L = 8
  numHashes: 8,       // k = 8 (每表哈希函数数)
  bucketWidth: 4,     // w = 4 (汉明距离阈值)
  vectorCount: 100000 // n = 100K
};
```

### 2.2 假阳性率理论计算

#### 步骤 1: 汉明距离 < w 的碰撞概率

对于 64bit SimHash，两个随机向量的汉明距离服从二项分布:

```
P(汉明距离 = d) = C(64, d) / 2^64

P(汉明距离 < 4) = Σ(d=0 to 3) C(64, d) / 2^64
                = [C(64,0) + C(64,1) + C(64,2) + C(64,3)] / 2^64
                = [1 + 64 + 2016 + 41664] / 1.84 × 10^19
                = 43745 / 1.84 × 10^19
                ≈ 2.37 × 10^-15
```

#### 步骤 2: 单表假阳性率

对于 n=100K 向量，单表假阳性率:

```
P(单表FP) ≈ 1 - (1 - p)^n
         ≈ 1 - (1 - 2.37×10^-15)^100000
         ≈ 2.37 × 10^-10
```

#### 步骤 3: 多表联合假阳性率

```
P(联合FP) = 1 - (1 - P(单表FP))^L
         = 1 - (1 - 2.37×10^-10)^8
         ≈ 1.90 × 10^-9
         = 0.00000019%
```

### 2.3 理论结论

| 指标 | 理论值 |
|------|--------|
| 单点汉明碰撞概率 | ~2.37 × 10^-15 |
| 单表假阳性率 | ~2.37 × 10^-10 |
| 8表联合假阳性率 | ~1.90 × 10^-9 (0.00000019%) |
| **与目标对比** | **远低于 0.1%** ✅ |

> **数学结论**: 理论假阳性率约为 **1.9 × 10^-9**，即 **19亿分之一**，远低于 0.1% 目标。
