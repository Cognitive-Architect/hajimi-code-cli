## 2. Recall测试方法论

**Ground Truth定义**: Layer 0全扫描暴力搜索 = 精确最近邻结果集，作为评估基准。

**评估指标公式**: Recall@k = |ANN结果 ∩ GroundTruth| / |GroundTruth|

**测试数据集生成**:
- 规模: 100~1000条384维向量（可扩展至10K）
- 分布: 高斯随机+Numpy标准化
- 种子: 固定随机种子保证可复现

**红线标准**:
- Recall@10 ≥90%（Top-10召回率红线）
- Recall@50 ≥95%（Top-50召回率目标）

---

## 3. O(log N)性能验证

**规模梯度测试**:
| 节点数 | 测试类型 | 目标延迟 |
|--------|----------|----------|
| 100 | 基准 | <1ms |
| 1K | 验证 | <3ms |
| 10K | 压力 | P50<5ms, P99<10ms |
| 50K | 极限 | P99<20ms |

**复杂度对比**:
- ANN搜索: O(log N * M * D)，D=384, M=16
- 暴力扫描: O(N * D)，线性增长

**延迟测量点**: P50, P95, P99百分位数，排除冷启动热身后采样。

## 4. ef_search调优策略

**参数范围**: ef_search ∈ [k, 10k]，k为返回数量。参数优化（parameter opt）在此范围调优。

**Recall-ef曲线**: ef=[k,2k,4k,8k,16k,32k,64k,128k,256k]，每值运行100次查询。绘制Recall-ef curve评估质量。

**推荐默认值**:
| 场景 | ef_search | 预期Recall |
|------|-----------|------------|
| 默认 | max(k, 16) | ~92% |
| 高精度 | 64 | ~96% |
| 极端精度 | 128/256 | ~99% |

## 5. 10K节点压力测试方案（10K node pressure test）

**批量插入（batch insert）TPS测试**:
- 批次: 1000条/批次，共10批次
- 测量: 每批次插入耗时，计算平均TPS
- 目标: TPS >500（含SQLite持久化）

**内存监控（memory RSS）**:
- 指标: Peak RSS（常驻内存峰值）
- 工具: `/usr/bin/time -v`或系统API
- 上限: 200MB@10K节点（含SQLite缓存）

**并发测试**: 10并发线程各执行100次查询，验证读事务隔离、无死锁。

## 8. 失败处理

**低Recall分析流程**:
1. 检查ef_search设置，尝试增大至64/128
2. 验证ground truth正确性（暴力扫描全遍历）
3. 检查数据分布，高维稀疏数据Recall天然受限
4. 回退方案: 启用Layer 0精确扫描（O(N)）

**性能退化回退**:
- 检测: P99延迟>10ms@10K节点
- 策略: 降级到单层Layer 0搜索
- 告警: 记录日志，触发Week 36优化任务
