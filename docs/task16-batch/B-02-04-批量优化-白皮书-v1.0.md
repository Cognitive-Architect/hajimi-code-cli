## 第1章：设计目标

### 1.1 性能目标

- **吞吐**: > 1000 ops/s
- **延迟**: < 100ms (p99)
- **数据一致性**: 崩溃零丢失

### 1.2 技术选型

| 技术 | 用途 | 理由 |
|:---|:---|:---|
| WAL | 写前日志 | 保证持久化 |
| 批量写入 | 100条事务 | 减少IO次数 |
| Gzip压缩 | 存储优化 | 减少磁盘空间 |
| 异步刷盘 | 非阻塞 | 提高吞吐 |

## 第2章：架构实现

### 2.1 核心类结构

```javascript
class BatchWriterOptimized {
  writeQueue[]      // 写入队列
  walBuffer[]       // WAL缓冲区
  
  async write()     // 入队
  async _flushBatch() // 批量刷盘
  async _recoverFromWAL() // 崩溃恢复
}
```

### 2.2 写入流程

```
write(key, value)
  ├── 入队 writeQueue
  ├── 入队 walBuffer
  └── 达batchSize? → _flushBatch()
                        ├── 写WAL
                        ├── 写主存储
                        └── 清空walBuffer
```

### 2.3 崩溃恢复

```
启动 → 检查WAL文件 → 重放未提交数据 → 清空WAL
```
