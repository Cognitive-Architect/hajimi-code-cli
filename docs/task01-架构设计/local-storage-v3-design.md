## 3. 关键技术决策

### 3.1 分片策略: Hash 前缀水平分片

```javascript
// 路由算法: SimHash-64 高 8bit → 16 分片
shard_id = (simhash_hi >> 56n) % 16  // 0-15
```

**选择理由**:
- ✅ 查询模式匹配: SimHash 查询天然单分片
- ✅ 并发性能: 16 独立锁，真并行读写
- ✅ 数据均匀: Hash 随机性避免热点

### 3.2 存储格式: .hctx v3

```
┌─────────────────────────────────────────────────────────────┐
│ Header (128 bytes)                                          │
│   - 魔数: 0x48415458 (HCTX)                                  │
│   - 版本: 3                                                  │
│   - SimHash-64 (16 bytes)                                   │
│   - 数据大小 + SHA256 校验                                   │
├─────────────────────────────────────────────────────────────┤
│ Metadata (JSON)                                             │
│   - 创建时间、标签、访问统计                                  │
├─────────────────────────────────────────────────────────────┤
│ Payload (实际数据)                                           │
│   - 大文件自动分块 (>1MB)                                    │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 同步策略: WebRTC + 文件导出降级

```
状态机: IDLE → DISCOVERING → CONNECTING → CONNECTED
                              ↓ (ICE_FAILED/TIMEOUT)
                         FILE_EXPORT (自动降级)
```
