## 2. 架构审计

### 2.1 级联哈希设计评估

```
内容块 ──→ SimHash-64 ──→ 汉明距离 < 3? ──→ MD5-128 ──→ 最终判定
                │               │                │
                ↓               ↓                ↓
             快速筛选        候选集(~27%)      精确裁决
```

**审计意见**: ✅ **设计合理**

| 层级 | 功能 | 冲突概率 | 评估 |
|:---|:---|:---|:---|
| SimHash | 粗筛 | 假阳性率 27% | 符合 LSH 理论预期 |
| MD5 | 精确校验 | 2⁻¹²⁸ ≈ 2.9×10⁻³⁹ | 密码学安全级别 |
| **级联合成** | 双重保障 | **≤7.98×10⁻³⁹** | **远低于 10⁻³⁰ 目标** |

**数学验证**: 独立事件联合概率公式正确应用
```
P_cascade = P(SimHash 假阳性) × P(MD5 碰撞)
          ≈ 0.27 × 2.9×10⁻³⁹
          ≈ 7.98×10⁻³⁹
```

### 2.2 ChunkHashV2 结构体审计

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│  simhash    │    md5      │   length    │    seed     │
│   64 bits   │  128 bits   │  32 bits    │  32 bits    │
│   8 bytes   │  16 bytes   │   4 bytes   │   4 bytes   │
└─────────────┴─────────────┴─────────────┴─────────────┘
总占用: 32 字节 (v1 的 4 倍)
```

**字段设计评估**:

| 字段 | 用途 | 审计意见 |
|:---|:---|:---|
| `simhash` (64bit) | 第一层快速粗筛 | ✅ 汉明距离 O(1) 计算，合理 |
| `md5` (128bit) | 第二层精确校验 | ⚠️ 密码学弱点已知，已规划升级路线 |
| `length` (32bit) | 防长度扩展攻击 | ✅ 安全加固措施 |
| `seed` (32bit) | 消除硬编码 SPOF | ⚠️ 作用域需明确 (RISK-H-004) |

### 2.3 存储格式版本控制

| hash_type | 格式 | 状态 | 兼容性 |
|:---|:---|:---|:---|
| 0x01 | Legacy SimHash-64 | 只读兼容 | ✅ 新读旧 |
| 0x02 | SimHash + MD5 (当前) | 默认 | ⚠️ 旧读新 Fail-fast |
| 0x03 | SimHash + SHA256 | v3.1 计划 | 🗓️ 规划中 |
| 0x04 | SimHash + BLAKE3 | v3.5 计划 | 🗓️ 规划中 |

**审计意见**: ✅ **版本控制策略合理**
- Fail-fast 原则确保旧读取器不会误读新格式
- 魔数变更 (`HCX2` vs `HCTX`) 提供明确区分
- 升级路线清晰：MD5 → SHA256 → BLAKE3

## 5. 风险审计

### 5.1 已修复风险验证

#### RISK-H-001: JS `uint64` 精度丢失 ✅

**问题**: `Number(buf.readBigUInt64LE(0))` 导致 64 位截断

**修复验证**:
```typescript
// ✅ 正确实现
const simhash = buf.readBigUInt64LE(0);  // 保持 bigint 类型

// ❌ 已禁止
const simhash = Number(buf.readBigUInt64LE(0));  // CI 断言会拦截
```

**审计结论**: ✅ **修复彻底**
- 类型定义 `SimHashValue = bigint` 贯穿全链路
- popcnt 实现采用分段计算策略（高32位 + 低32位）
- CI 断言 `assertSimHashType()` 防止回退

#### RISK-H-002: 旧版本兼容性问题 ✅

**问题**: 旧读取器无法识别新格式的 24B 条目

**修复验证**:
```typescript
// 魔数变更
const SUPPORTED_MAGICS = ['HCX2'];  // 新格式
// 旧格式 'HCTX' 不在支持列表

// 最小兼容版本检查
if (header.minCompatibleVersion > READER_VERSION) {
  return { success: false, error: { code: HctxErrorCode.MIN_VERSION_TOO_HIGH } };
}
```

**审计结论**: ✅ **Fail-fast 策略正确实施**
