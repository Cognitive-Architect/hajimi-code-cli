#### Q2: 失败原因是实现缺陷还是测试环境问题？

**根因诊断：实现设计问题（数据读取不一致）**

**问题定位**（代码级分析）：

```javascript
// src/security/rate-limiter-sqlite-luxury.js

// saveBucket: 异步入队，不立即写入数据库
async saveBucket(ip, tokens, lastRefill) {
  this.writeQueue.push({ ip, tokens, lastRefill }); // 仅入队
  if (this.writeQueue.length >= this.config.batchSize) {
    await this._flushBatch(); // 达到阈值才刷盘
  }
}

// getBucket: 直接从数据库读取（同步）
getBucket(ip) {
  const stmt = this.stmtCache.get('getBucket');
  // ... 从DB读取 ...
  // 问题：读不到队列中待写入的数据！
}

// checkLimit: 先getBucket（读DB），后saveBucket（入队）
async checkLimit(ip, tokens = 1) {
  let bucket = this.getBucket(ip); // 读DB（可能读到旧数据）
  // ... 计算 ...
  await this.saveBucket(ip, bucket.tokens, bucket.lastRefill); // 入队
}
```

**核心问题**：
- `saveBucket` 写入队列（异步批量）
- `getBucket` 从数据库读取（同步）
- **数据不一致**：队列中的新数据，`getBucket`读不到

**影响测试**：
1. **LUX-BASE-013**: `saveBucket` → `_flushBatch` → `getBucket`，但`getBucket`可能读不到刚写入的数据（时序问题）
2. **LUX-BASE-016**: 循环调用`checkLimit`，每次`getBucket`读DB，但之前的`saveBucket`数据在队列中未刷盘，导致重复计算capacity
3. **BONUS**: 批量写入后`getBucket`返回null（数据还在队列中）

### 要素3：落地可执行路径（修复处方）

#### 修复方案：`getBucket` 优先读取队列中最新数据

**修改文件**: `src/security/rate-limiter-sqlite-luxury.js`

**修改位置**: `getBucket` 方法（第185-201行）

**修改前**:
```javascript
getBucket(ip) {
  const stmt = this.stmtCache.get('getBucket');
  stmt.bind([ip]);
  
  const result = stmt.step();
  if (result) {
    const row = stmt.getAsObject();
    stmt.reset();
    return {
      tokens: row.tokens,
      lastRefill: row.last_refill
    };
  }
  
  stmt.reset();
  return null;
}
```

**修改后**:
```javascript
getBucket(ip) {
  // 修复：优先检查队列中是否有待写入的最新数据
  const pendingItem = this.writeQueue.findLast(item => item.ip === ip);
  if (pendingItem) {
    return {
      tokens: pendingItem.tokens,
      lastRefill: pendingItem.lastRefill
    };
  }
  
  // 队列中没有，从数据库读取
  const stmt = this.stmtCache.get('getBucket');
  stmt.bind([ip]);
  
  const result = stmt.step();
  if (result) {
    const row = stmt.getAsObject();
    stmt.reset();
    return {
      tokens: row.tokens,
      lastRefill: row.last_refill
    };
  }
  
  stmt.reset();
  return null;
}
```

**备用方案**（如环境不支持`findLast`）：
```javascript
// 使用倒序查找
for (let i = this.writeQueue.length - 1; i >= 0; i--) {
  if (this.writeQueue[i].ip === ip) {
    return {
      tokens: this.writeQueue[i].tokens,
      lastRefill: this.writeQueue[i].lastRefill
    };
  }
}
```
