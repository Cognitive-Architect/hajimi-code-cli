## 2. 级联哈希方案 (核心解决方案)

### 2.1 设计原理

**两道闸门**: SimHash 粗筛 + MD5 精确校验

```
内容块 A ──→ SimHash ──→ 汉明距离 < 3? ──→ MD5 比较 ──→ 最终判定
                │                │                │
                ↓                ↓                ↓
             快速筛选         候选集           精确裁决
```

### 2.2 概率模型

| 层级 | 事件 | 概率 |
|:---|:---|:---|
| SimHash 层 | A = 假阳性 | P(A) = 0.27 (实测) |
| MD5 层 | B = 碰撞 | P(B) = 2⁻¹²⁸ ≈ 2.9×10⁻³⁹ |
| **级联合成** | A∩B | **P_cascade ≤ 7.98×10⁻³⁹** |

**结论**: 7.98×10⁻³⁹ ≪ 10⁻³⁰ (宇宙级安全目标)，**达标**。

## 3. ChunkHashV2 结构体

```
总占用: 32 字节 (v1 的 4 倍)

┌─────────────┬─────────────┬─────────────┬─────────────┐
│  simhash    │    md5      │   length    │    seed     │
│   64 bits   │  128 bits   │  32 bits    │  32 bits    │
│   8 bytes   │  16 bytes   │   4 bytes   │   4 bytes   │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

### 字段说明

| 字段 | 位数 | 用途 |
|:---|:---|:---|
| `simhash` | 64 | 第一层：快速粗筛，O(1) 汉明距离计算 |
| `md5` | 128 | 第二层：精确校验，冲突率 2⁻¹²⁸ |
| `length` | 32 | 防长度扩展攻击，支持 2KB-16KB 可变块 |
| `seed` | 32 | 消除硬编码 SPOF，支持可配置哈希 |

## 7. 关键技术决策

### 7.1 汉明距离阈值: h=3

| 阈值 | 假阳性率 | 召回率 | 评估 |
|:---|:---|:---|:---|
| h=2 | ~13.5% | ~85% | 边缘相似块漏检风险 |
| **h=3** | **27%** | **~98%** | **当前最优** |
| h=4 | ~45% | ~99.5% | 边际增益，过滤效率下降 |

**结论**: 保持 h=3，不调整。

### 7.2 放弃方案

| 方案 | 排除原因 |
|:---|:---|
| 增加哈希位数 (64→128) | 无法消除 LSH 特性；成本过高 |
| 更换 LSH 变体 (MinHash) | CDC 场景以序列相似度为主，收益有限 |
| 全新 CDC 专用哈希算法 | 18-36 人月，远超级联方案 2-4 人月 |
